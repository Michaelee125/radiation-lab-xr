# Radiation Lab XR 中文部署与测试指南

本指南适用于桌面程序包 `Radiation-Lab-XR-Quest`。程序运行时完全使用本地资源，不会加载 CDN、远程模型、字体、声音、统计服务或远程 API。

## 1. 程序包内容

```text
Radiation-Lab-XR-Quest/
├── client/                    可直接部署的静态网站
├── server/                    Cloudflare Worker 兼容入口（普通静态部署可忽略）
├── mac-test-server.mjs        Mac 本地测试服务器
├── 启动-Mac-测试.command      Mac 双击启动脚本
├── 部署与测试指南.md           本文件
└── README.md                  英文项目说明
```

部署时必须保持 `client` 内部的目录结构不变。不要直接双击 `client/index.html`，因为浏览器的模块和离线缓存需要通过 HTTP 或 HTTPS 服务器加载。

## 2. 在 Mac 上启动

### 方法 A：双击启动（推荐）

1. 双击程序包根目录的 `启动-Mac-测试.command`。脚本会优先使用 Node.js；如果没有 Node.js，会自动尝试使用 Mac 上的 Python 3。
2. 如果脚本提示两者都不可用，请安装 Node.js 20 或更高版本后重试。
3. 脚本会启动本地服务器，并自动打开：

   `http://127.0.0.1:4173`

4. 测试结束后，回到打开的“终端”窗口，按 `Control + C` 停止服务器。

如果 macOS 第一次阻止脚本运行，请按住 Control 点击该文件，选择“打开”，再确认一次。也可以进入“系统设置 → 隐私与安全性”，允许这次打开。

### 方法 B：使用终端启动

打开“终端”，进入程序包中的 `client` 目录，然后运行：

```sh
node ../mac-test-server.mjs
```

浏览器访问 `http://127.0.0.1:4173`。本机 `localhost`/`127.0.0.1` 可用于桌面开发和 Service Worker 测试，但不能代替 Quest 所需的 HTTPS 部署。

## 3. Mac 桌面测试方法

建议使用最新版 Chrome 或 Edge 进行测试。

### 基本功能

1. 页面完整显示后，等待右上角及场景内出现 **Offline ready**。
2. 分别测试 Alpha、Beta、Gamma，并测试所有七种非空组合。
3. 尝试关闭唯一启用的辐射类型，确认程序阻止“全部关闭”。
4. 用鼠标依次拖动纸、铝、铅三块板：
   - 板子横跨粒子束时，无需放入中央卡槽就应立即生效；
   - 按住鼠标拖动时，屏蔽材料名称、教学说明、粒子吸收位置和 GM 读数应实时变化；
   - 在源与 GM 管之间的不同横向位置松手，板子应留在该位置并继续生效；
   - 将板子拖出束路后松手，板子应回到材料架并恢复 `NO SHIELD`；
   - 换用另一块板时，上一块板应自动回架；
   - 靠近中央卡槽松手时，板子仍可吸附到卡槽。
   左键用于按钮和材料板；按住鼠标右键拖动，或使用方向键，可以左右/上下转动视野查看环绕式说明面板。
5. 测试 **Radiation visuals**：关闭后，辐射模型和轨迹应同时消失，但粒子运动计算、屏蔽效果、声音和 GM 读数仍继续；GM 来源行和说明面板应进入不泄露答案的 Mystery mode。重新打开后模型、轨迹和说明答案应恢复。
6. 测试 **Hide controls**：隐藏完整的 Source & Experiment Controls 面板，并确认独立的 **Show controls** 按钮仍可操作。
7. 测试 **Start Mystery**：一次操作应同时隐藏三种辐射模型、全部轨迹和最上方控制面板；**Exit Mystery** 按钮必须保留，并能完整恢复显示。
8. 确认 GM 的大型 0–100 CPS 指针表会在每次一秒计数变化时平滑转到新角度；高对比度的大号 CPS 数字无需靠近即可看清，并继续显示滚动平均值。
9. 选择 Gamma 和 Lead，确认所有伽马动画都在铅板处停止，GM 管主示数、指针、1 秒计数和滚动平均值立即变为 `0`。在本考纲化简模型中，Lead 状态会省略本底计数。

### 控制台检查

1. 打开开发者工具（Chrome/Edge：`Option + Command + I`）。
2. 查看 Console，确认没有红色阻塞错误。
3. 查看 Network，确认运行时没有访问 CDN、远程字体、声音、模型或 API。

### Mac 离线测试

1. 在线打开一次页面，等待 **Offline ready**。
2. 打开开发者工具的 Network 面板，将网络条件设为 **Offline**。不要勾选“Disable cache”。
3. 刷新页面，确认实验室仍能完整加载并可以操作。
4. 测试完成后把网络条件恢复为 **No throttling**。

如果更新程序后仍显示旧版本，请关闭旧页面并重新打开；仍有问题时，在开发者工具的 Application/Storage 中清除此站点数据，再重新加载并等待离线缓存完成。

## 4. 部署到 HTTPS 网站

### 推荐方法：静态网站托管

本程序没有后端数据库，可部署到任何支持 HTTPS 的静态网站服务，例如学校 HTTPS 网站、Cloudflare Pages、Netlify 或 GitHub Pages。

通用步骤：

1. 新建一个静态网站项目。
2. 将程序包中 **`client` 目录内的全部内容** 上传为网站根目录。
3. 不需要构建命令，不需要安装依赖，也不需要设置环境变量。
4. 确认网站首页文件是 `index.html`。
5. 确认托管服务提供有效、受信任的 HTTPS 证书。
6. 打开生成的 `https://...` 地址，确认页面出现 **Offline ready**。

如果平台要求填写“发布目录”，上传整个程序包时填写 `client`；如果平台允许直接上传文件夹，则直接上传 `client` 文件夹中的内容。

### 学校服务器部署

把 `client` 内全部文件复制到某个 HTTPS 站点目录，并保持相对路径不变。服务器应：

- 允许访问 `service-worker.js`；
- 将 `.webmanifest` 作为 `application/manifest+json` 提供；
- 不把 JavaScript 模块改写成 HTML；
- 为整个站点使用 Quest 信任的 HTTPS 证书。

Quest 上不能使用 Mac 的 `http://192.168.x.x:4173` 地址进入 WebXR。`localhost` 的安全上下文例外只适用于运行浏览器的那台设备。要从 Quest 访问 Mac 局域网服务，仍需使用 Quest 信任的 HTTPS 域名和证书；课堂使用更推荐稳定的 HTTPS 静态托管。

## 5. Meta Quest 测试方法

测试前记录 Quest 型号、系统版本、Quest Browser 版本、HTTPS 地址和测试日期。

### 首次在线测试

1. Quest 连接 Wi-Fi，在 Quest Browser 中打开部署后的准确 HTTPS 地址。
2. 等待浏览器界面和场景内都显示 **Offline ready**。
3. 点击 **Enter VR** 进入沉浸模式。
4. 确认初始视角已经靠近实验台，不需要向前走或突破 Guardian 边界；源、材料架和 GM 管位于中央。向左转头查看放大的说明与图例面板，向右转头查看大型 GM 指针表，三者以悬浮弧形环绕观察者且不遮挡实验台。
5. 分别使用左右控制器射线点击 Alpha、Beta、Gamma、Radiation visuals、Sound、No shield 和 Reset。
6. 隐藏和重新显示 Source & Experiment Controls 面板，确认隐藏后仍可通过独立按钮恢复。
7. 开启 Mystery Mode，确认辐射模型、轨迹和顶部控制面板一起消失；退出后全部恢复。
8. 指向纸板并按住扳机，沿粒子束缓慢横向拖动：
   - 仍按住扳机时，屏蔽状态和粒子表现应实时变化；
   - 在中央卡槽以外的位置松开，纸板应留在束路并继续生效；
   - 拖出束路后，纸板应回到材料架。
9. 对铝板和铅板重复上述操作，并测试材料替换。
10. 测试所有七种辐射组合、Mystery Mode、声音许可、GM 指针波动和 Reset。
11. 坐姿和站姿各测试一次，确认放大的数字与指针表清晰、按钮可达、没有不舒适的镜头运动或闪烁。

### Quest 离线测试

1. 保持在线页面打开，直到 **Offline ready** 已明确显示。
2. 退出 VR 并关闭该浏览器页面。
3. 把页面加入书签；如果当前 Quest Browser 提供安装网页应用/添加到主屏幕选项，也可以安装。
4. 关闭 Quest 的 Wi-Fi。
5. 从同一个书签、已安装入口或完全相同的 URL 重新打开。
6. 确认实验室离线加载，并再次测试：
   - 一个辐射切换按钮；
   - 一块材料板的实时拖动；
   - Mystery Mode；
   - Hide/Show controls；
   - GM 读数；
   - Reset；
   - 进入 VR。
7. 测试完成后恢复 Wi-Fi，以便以后接收程序更新。

首次课堂使用前务必在目标 Quest 上完成一次完整在线缓存和一次断网重开。Quest Browser 或系统更新后，也建议重新执行离线测试。

## 6. 更新部署

发布新版本时：

1. 用新的 `client` 内容完整替换服务器上的旧文件。
2. `service-worker.js` 中的缓存版本必须递增；本程序包已经包含当前版本。
3. 在线打开网站，等待更新缓存完成。
4. 关闭并重新打开页面后，再执行一次离线测试。

不要在上课过程中强制所有头显刷新。建议课前完成更新、缓存和断网验证。

## 7. 常见问题

### Quest 看不到 Enter VR 或无法进入 VR

- 确认地址以 `https://` 开头；
- 确认证书没有警告且被 Quest 信任；
- 确认使用 Quest Browser，而不是把 HTTP 页面嵌入其他应用；
- 关闭旧标签后重新打开准确地址。

### 材料板没有生效

- 板子必须真正横跨可见的三条粒子束，而不是只靠近设备；
- 沿束路平面缓慢拖动，观察 `LIVE SHIELD` 标签是否改变；
- 使用 Reset 后重新测试。

### 离线时打不开

- 必须先在线等待 **Offline ready**；
- 离线重开时必须使用同一个站点和路径；
- 浏览器清理存储后需要重新联网缓存；
- 首次安装或更新后，应在断网前先关闭并重新打开一次。

### Mac 双击脚本后立即关闭

- 在终端运行 `node --version`，确认已安装 Node.js 20 或更高版本；
- 确认 4173 端口没有被其他程序占用；
- 改用“方法 B”启动，可以在终端看到具体错误信息。
